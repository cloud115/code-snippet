# JavaScript 代码片段库 - 开发流程指南

## 开发流程指南

### 分支管理
- **main**：主分支，用于发布稳定版本
- **feature/**：功能分支，用于开发新功能
- **bugfix/**：修复分支，用于修复bug

### 提交规范
- **提交信息格式**：`类型: 描述`
- **类型包括**：
  - feat：新功能
  - fix：修复
  - docs：文档
  - style：样式
  - refactor：重构
  - test：测试
  - chore：构建
- **描述要求**：简洁明了，说明变更的内容和原因

### 版本管理
- 使用语义化版本控制（Semantic Versioning）
- 版本号格式：`MAJOR.MINOR.PATCH`
  - **MAJOR**：主版本号，当你做了不兼容的API修改时递增
  - **MINOR**：次版本号，当你添加了向后兼容的功能时递增
  - **PATCH**：补丁版本号，当你做了向后兼容的bug修复时递增

### 代码审查
- 所有代码变更建议经过代码审查
- 代码审查重点关注：
  - 功能正确性
  - 代码质量和可读性
  - 性能优化
  - 安全性
