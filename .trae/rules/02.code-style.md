# JavaScript 代码片段库 - 代码风格指南

## 代码风格指南

### 命名规范
- **文件命名**：使用数字+功能名称的格式，如 `01.Observer.js`
- **变量和函数**：使用驼峰命名法（camelCase）
- **类名**：使用帕斯卡命名法（PascalCase）
- **常量**：使用全大写字母和下划线（UPPERCASE_WITH_UNDERSCORES）

### 代码格式
- **缩进**：使用2个空格
- **分号**：必须使用
- **引号**：使用单引号
- **行尾符**：必须使用Unix风格LF（\n），禁止使用Windows风格CRLF（\r\n）
- **行宽**：建议不超过120个字符
- **空行**：函数之间、代码块之间使用空行分隔
- **括号**：使用大括号包围代码块，即使只有一行代码

### 注释规范
- **文件头部**：添加文件功能描述
- **函数和方法**：添加JSDoc注释，包含功能描述、参数说明和返回值说明
- **复杂逻辑**：添加内联注释，解释关键逻辑和算法
